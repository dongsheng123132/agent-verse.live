# 预测市场设计说明（参考 Polymarket）

## 一、当前实现 vs Polymarket

| 维度 | 我们（AgentVerse） | Polymarket |
|------|-------------------|------------|
| **身份** | 积分账户（钱包/游客），积分即「筹码」 | 钱包 + USDC，真金白银 |
| **题目** | 多选一（如：哪个节目最火？A/B/C）或二选一（是/否） | 多为二选一（Yes/No），价格 0–1 表示概率 |
| **下注** | 选一个选项、扣 N 积分；猜对返还 + 奖励倍数（如 1.5x） | 买 Yes/No 份额，结算时正确方 1 份=1 美元 |
| **流动性** | 无 AMM，固定赔率（我们定） | AMM 做市，价格由买卖推动 |
| **结算** | 运营/管理员调用「结算」接口指定正确选项 | 链上预言机或委员会决议 |

结论：我们是**积分制、固定赔率、多选/二选**的轻量预测；Polymarket 是**法币稳定币 + 概率市场 + AMM**。可借鉴的是：**题目表述方式、二选一/多选结构、结算规则写清楚**。

## 二、题目设计建议（2026 春晚）

- **可验证**：结算依据明确（如「微博/YouTube 点赞数 24h 最高」「是否进热搜前十」）。
- **选项可控**：单题选项不宜过多（建议 ≤ 6），否则积分分散、体验差。
- **可动态**：题目与选项可后台/配置更新，不必写死（见下「题目可更新机制」）。

### 示例题目（可直接作种子或后台配置）

1. **最受欢迎节目**（多选一）  
   选项：岳云鹏相声 / 沈腾小品 / 周深独唱 / AI·数字人表演 / 分会场互动 / 其他。  
   结算：官方或第三方公布的「最受欢迎」或 24h 点赞最高。

2. **哪位艺人节目热度最高？**（多选一）  
   选项：岳云鹏 / 沈腾 / 刘宇宁 / 周深 / 丁真 / 其他。

3. **AI/数字人相关节目是否进全网热搜前十？**（二选一）  
   选项：是 / 否。

4. **最受欢迎分会场**（多选一）  
   选项：北京主会场 / 哈尔滨 / 义乌 / 合肥 / 宜宾。

节目候选可参考你整理的清单（歌舞、小剧场、明星、科技、互动等），**题目与选项建议可配置、可增删**，便于随官方/网传名单更新。

## 三、题目可更新机制（避免写死）

- **后端**：题目与选项存库或配置文件，提供「题目列表」「单题详情」接口；管理员可「新增/编辑/关闭」题目、设置选项与结算规则。
- **结算**：到时间点后调用「结算」接口，传入题目 id + 正确选项 id，系统按规则给猜中用户加积分（如 1.5 倍返还）。
- **前端**：预测市场页只消费「题目列表 + 选项」，不写死 50 个候选；若后续做「50 位候选模板」，可做成一条题目 + 50 个选项的**单题**，或拆成多题（如按类型：歌舞 / 小剧场 / 明星 / 科技 各一题）。

---

## 四、与 Polymarket 的对照（可借鉴点）

- **二选一**：很多题可拆成「是否…」形式，和 Polymarket 一致，理解成本低。
- **题目文案**：一句说清「预测什么」「何时、按什么标准结算」。
- **结算透明**：在题目说明里写「结算依据：…」「结算时间：…」。

当前代码位置：题目种子在 `server/local-api.ts` 的 `predictionTopics`；结算在 `POST /api/v1/predictions/settle`。要支持「题目可实时更新」只需把 `predictionTopics` 改为从数据库或配置读取即可。
